# RestControllerAdvice의 장점 및 없을 경우의 불편함

## ✅ RestControllerAdvice의 장점

### 1. 전역 예외 처리 가능 (Global Exception Handling)
- 모든 컨트롤러에서 발생하는 예외를 한 곳에서 처리할 수 있다.
- 예외 응답 형식을 일관되게 제공할 수 있어 유지보수가 쉬워진다.

### 2. 공통 로직 분리
- 로깅, 공통 응답 포맷, 공통 헤더 설정 등을 컨트롤러 밖으로 분리할 수 있다.
- 컨트롤러 코드가 간결해지고 가독성이 좋아진다.

### 3. 유지보수성 향상
- 예외 처리 정책을 수정할 때 한 파일만 수정하면 전체 API에 반영된다.
- 프로젝트 규모가 커질수록 효과가 커진다.

### 4. 가독성 증가
- 컨트롤러에는 핵심 비즈니스 로직만 남고, 예외 처리는 Advisor에서 전담한다.
- 코드 구조가 더 명확해진다.

---

## ❌ RestControllerAdvice가 없을 때의 불편함

### 1. 컨트롤러마다 try-catch 반복
- 모든 컨트롤러에서 중복된 예외 처리 코드가 발생한다.
- 코드가 지저분해지고 유지보수가 어려워진다.

### 2. 응답 포맷 일관성 부족
- 일부 API는 JSON, 일부는 문자열, 일부는 500 에러 등 제각각이 될 수 있다.
- API 품질이 낮아지고 사용자 혼란이 생긴다.

### 3. 유지보수 어려움
- 정책 변경 시 모든 컨트롤러의 예외 처리를 일일이 수정해야 한다.
- 규모가 커질수록 관리가 불가능해진다.

### 4. 공통 처리의 어려움
- 로깅, 인증 실패 처리 등 공통 로직이 각 컨트롤러에 분산된다.
- 중복 코드 증가 + 누락 가능성 증가.
